# Practica1 - Flujo Básico de git y github
Primera Practica en un repositorio nuevo


# 1 ¿Qué es una Base de Datos Relacional?
Es una forma de almacenar información de manera ordenada y estructurada.
Los datos se guardan en tablas que pueden relacionarse entre sí.
Esto permite conectar información sin necesidad de repetirla muchas veces.
Gracias a estas relaciones, los datos son más fáciles de entender y manejar.
También ayuda a mantener la información consistente y bien organizada.
Es muy usada en sistemas como páginas web, aplicaciones y sistemas empresariales.

# 2 ¿Qué es una Tabla en una Base de Datos Relacional?
Una tabla es donde realmente se almacenan los datos dentro de la base de datos.
Está formada por columnas y filas, similar a una hoja de Excel.
Cada tabla representa algo específico, como usuarios, productos o ventas.
Ayuda a mantener la información separada y ordenada.
Todas las tablas tienen una estructura definida desde el inicio.
Gracias a esto, los datos son más fáciles de consultar y actualizar.

# 3 Qué es una relación en Bases de Datos Relacionales?
Una relación es la conexión que existe entre dos o más tablas.
Se usa para vincular información que está distribuida en diferentes tablas.
Normalmente se crea usando una clave primaria y una clave foránea.
Esto evita repetir datos innecesarios en varias tablas.
También ayuda a mantener la información más clara y organizada.
Un ejemplo común es la relación entre clientes y sus pedidos.

# 4 ¿Qué representa una columna?
Una columna representa un tipo específico de dato dentro de una tabla.
Define qué información se va a guardar, como nombre, edad o correo.
Cada columna tiene un nombre y un tipo de dato definido.
Todas las filas deben tener datos del mismo tipo en esa columna.
Esto ayuda a mantener un orden y evitar errores en la información.
Las columnas describen las características de cada registro.

# 5 ¿Qué representa un renglón?
Un renglón representa un registro completo dentro de la tabla.
Contiene toda la información de un solo elemento.
Por ejemplo, todos los datos de un cliente específico.
Cada renglón suele identificarse con un valor único.
Esto permite diferenciarlo fácilmente de los demás registros.
En conjunto, los renglones forman el contenido principal de la tabla.

# 6 ¿Qué es un motor de base de datos relacional?
Es el software que se encarga de manejar la base de datos.
Permite guardar, buscar, modificar y eliminar información.
También se encarga de la seguridad de los datos.
Administra el acceso cuando varios usuarios trabajan al mismo tiempo.
Ejecuta las consultas hechas con SQL.
Sin el motor, la base de datos no podría funcionar.

# 7 ¿Qué es un motor de base de datos relacional?
MySQL es muy usado en aplicaciones web y proyectos pequeños.
PostgreSQL es conocido por ser muy completo y confiable.
Oracle Database se usa mucho en grandes empresas.
SQL Server es popular en entornos de Microsoft.
MariaDB es una alternativa moderna y de código abierto.
Todos ellos permiten trabajar con bases de datos relacionales.

# 8 ¿Cuál es el motor de base de datos ligero que no necesita un servidor?
SQLite es un motor de base de datos muy liviano.
No necesita un servidor para funcionar.
Toda la base de datos se guarda en un solo archivo.
Es fácil de usar y configurar.
Se utiliza mucho en aplicaciones móviles y de escritorio.
Es ideal para proyectos pequeños o de prueba.

# 9 Qué significa SQL en las bases de datos relacionales?
SQL significa Lenguaje de Consulta Estructurado.
Es el lenguaje que se usa para comunicarse con la base de datos.
Permite crear y modificar tablas.
También sirve para consultar información almacenada.
Con SQL se pueden actualizar y eliminar datos.
Es un estándar usado en casi todos los motores relacionales.

# 10 ¿Cuáles son los elementos de un sistema de información basado en bases de datos relacionales?
El primer elemento es la base de datos donde se guardan los datos.
También está el motor de base de datos que la administra.
Se utiliza SQL para interactuar con la información.
Existe una aplicación que permite a los usuarios acceder a los datos.
Los usuarios forman parte importante del sistema.
Todo funciona sobre un equipo y un sistema operativo.
